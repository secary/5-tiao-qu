# TODO

本文件记录 `5-tiao-qu` 当前 MVP 阶段的开发任务。优先级围绕一条主链路展开：

上传规则书 -> 解析文本/OCR -> 切分规则片段 -> 建立索引 -> 提问检索 -> 基于引用回答 -> 回看出处

  - `.env.example`。

## P1：规则文本切分

- [ ] 实现基础文本清洗：
  - 去除重复页眉页脚。
  - 合并断行。
  - 保留编号、标题和关键术语。
- [ ] 按页码、章节和段落切分规则片段。
- [ ] 控制 chunk 大小，避免过碎或过大。
- [ ] 每个 chunk 必须保留来源页码和原文。
- [ ] 为跨页规则保留相邻上下文字段或检索扩展策略。

## P1：Embedding 与向量检索

- [ ] 设计 embedding provider 接口，避免绑定单一供应商。
- [ ] 为每个 `rule_chunk` 生成 embedding。
- [ ] 使用 pgvector 建立向量索引。
- [ ] 实现按问题检索相关规则片段。
- [ ] 检索结果返回：
  - chunk ID
  - 相似度分数
  - 页码
  - 章节标题
  - 原文片段
  - 来源类型和优先级

## P1：RAG 问答接口

- [ ] 实现提问 API。
- [ ] 对用户问题生成 embedding 并检索相关 chunk。
- [ ] 构建 RAG prompt，明确要求模型：
  - 只使用提供的规则片段。
  - 不知道就说不知道。
  - 用用户提问的语言回答。
  - 先给结论，再给依据。
  - 每条依据绑定引用 ID。
- [ ] 统一回答类型：
  - `direct_rule`
  - `related_inference`
  - `not_found`
  - `conflict`
- [ ] 检索不到足够依据时，返回“规则书中未找到明确说明”。
- [ ] 回答结果必须带 citations。
- [ ] 当多个来源冲突时，展示冲突来源，不静默合并。
